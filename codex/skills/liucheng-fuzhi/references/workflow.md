# Liucheng Fuzhi Workflow Reference

## 1. Standard 5-step replication model

1. Clone the hot site's core mechanic.
2. Change the UI enough to feel like a new product.
3. Add ad inventory and private-traffic capture.
4. Ship and run the first cold-start distribution wave.
5. Measure, refine, and save the pipeline as a reusable template.

## 2. Minimal config checklist

- Production domain
- CTA target URL
- CTA copy
- Ad provider code or affiliate links
- Analytics script or event bridge
- Social sharing copy

## 3. KPI priorities

- UV
- Start rate
- Completion rate
- Share rate
- Private CTA click rate
- Ad click rate

Interpretation:
- Low start rate: fix hero section and first-screen copy
- Low completion rate: reduce friction in the interaction flow
- Low share rate: improve result-page ego payoff
- Low private click rate: improve CTA clarity and incentive

## 4. Recommended delivery pattern

When building these sites, aim to ship:
- one static site folder,
- one config surface in JS,
- one result page with share/export/capture,
- one deploy note,
- one short "tomorrow checklist"

## 5. Distribution heuristics

Prioritize fast, lightweight channels first:
- private groups
- Xiaohongshu
- Moments / friend circles
- comment replies on related posts
- short demo videos showing result reveal

Posting pattern:
- headline with result label
- one-line reaction
- screenshot of result page
- direct link

## 6. Template-saving rule

After each project, save:
- what was cloned,
- what was changed,
- which share hooks worked,
- which CTA converted,
- which metrics were weak,
- what should become the next starter template.
